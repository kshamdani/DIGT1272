WCAG 2.1 Framework

The Four POUR Principles

Every WCAG guideline traces back to four core principles. Content must be Perceivable, Operable, Understandable, and Robust.

Perceivable
WCAG Principle 1

Information and user interface components must be presented in ways that users can perceive, meaning no content should be invisible to all of a user's senses.

  • Text alternatives for all non-text content (images, icons)
  • Captions and audio descriptions for video
  • Content adaptable without losing information or structure
  • Sufficient colour contrast for text (4.5:1 minimum)
Operable
WCAG Principle 2

User interface components and navigation must be operable by all users, including those who cannot use a mouse or touchscreen.

  • All functionality available via keyboard alone
  • No keyboard traps. Users can always navigate away
  • No content that flashes more than three times per second
  • Skip navigation links and meaningful page titles
Understandable
WCAG Principle 3

Information and the operation of the user interface must be understandable, meaning users must be able to comprehend both the content and how to operate controls.

  • Page language declared in HTML for screen readers
  • Predictable navigation. No unexpected context changes
  • Error messages that identify and describe the problem clearly
  • Labels for all form inputs and controls
Robust
WCAG Principle 4

Content must be robust enough that it can be reliably interpreted by a wide variety of user agents, including current and future assistive technologies.

  • Valid, well-formed HTML to ensure correct parsing
  • Name, role, and value set for all UI components
  • Status messages communicated to assistive tech without focus
  • ARIA landmarks and roles used correctly
Guiding Questions

Questions That Drive This Project

Three core questions about digital citizenship, developer responsibility, and inclusive design underpin everything here.

01

"How much should developers be liable for ensuring that their digital services are available to people with different kinds of disabilities to use?"

02

"How could creating accessibility standards for regulating the internet, similar to other infrastructure, support and increase 'digital equality' and help all people to participate in digital communities?"

03

"How does using accessibility design enable people with disabilities to have a greater chance of participating in digital communities?"

Project Overview

Who Benefits & Why It Was Built

Accessibility is too often considered an available add-on rather than a necessary component of digital citizenship. With more governments, education, and essential services moving to online platforms, inaccessible design limits opportunities for full participation in digital society.

This project demonstrates that inclusive design provides equal access to information and strengthens digital communities. It is built for novice web developers, designers, and educators who may not be aware of the barriers created by inaccessible design.

Through social media, developer forums, and accessibility awareness events, this site can be shared as an interactive way for communities to learn and understand the importance of inclusive design choices.
